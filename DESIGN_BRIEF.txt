1. PRODUCT PURPOSE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Name:        CRMA Smart Academy (โรงเรียนนายร้อย)
Type:        LINE LIFF web app (also usable standalone in browser)
Platform:    LINE in-app webview (primary) + Safari iOS + Chrome Android + Desktop
Users:       Cadets, Instructors, Officers, Admins at Chulachomklao Royal Military Academy, Thailand
Language:    Thai (default) with English fallback
Deploy:      Vercel (preview + production)

Mission:
  Consolidate fragmented CRMA cadet systems into one LINE-distributed web app.
  Replace the previous Expo/React Native mobile app with a zero-install LIFF experience
  accessible via the official CRMA LINE Official Account rich-menu.

Core Problem Solved:
  Cadets currently switch between: Moodle (LMS), paper schedules, separate health apps,
  LINE group announcements, and physical grade reports. CRMA Smart Academy merges all
  of these into one authenticated single-page experience inside LINE.

11. DESIGN PRINCIPLES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Mobile-first, LIFF-aware — 100dvh, safe-area insets, no fixed 390px iPhone frame around production UI
2. Thai-first — all labels default Thai; EN fallback via next-intl
3. Dark hero cards — slate-900 backgrounds with amber accent glow for announcement/hero areas
4. Soft light cards — white bg, border-slate-100, shadow-sm for data/content cards
5. Amber = active/selected — nav items, progress bars, dot indicators, CTA buttons
6. Emerald = enrol/trust — auth CTAs, session-minted states
7. Consistent test IDs — every interactive element carries data-testid matching predecessor
8. No localStorage — httpOnly cookies only (LINE webview evicts localStorage)
9. Accessibility — role="tab" aria-selected, aria-label on icon buttons, role="alert" for toasts
10. Progressive — Phase 3-8 stub views = immediate deploy; content fills per phase without regression
