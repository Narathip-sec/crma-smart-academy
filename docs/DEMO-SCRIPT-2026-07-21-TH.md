# Demo Script — CRMA Smart Academy LIFF (5 นาที)

**ใช้ประกอบสไลด์ "Feature walkthrough"** — ดูโครงร่างสไลด์เต็มใน `docs/PRESENTATION-PREP-PLAN.md`
**อุปกรณ์:** มือถือจริงเปิดผ่าน LINE (ทดสอบแล้ว iOS; Android รอทดสอบ) ต่อจอ/แชร์สกรีนให้กรรมการเห็น
**ข้อมูลสาธิต:** ของจริงจาก DB (ไม่ใช่ mock) — ตารางเรียน/เมนูอาหารสัปดาห์ปัจจุบันของเดือนที่นำเสนอ (สร้างจาก `npm run demo:extend`), กิจกรรม 3 รายการ, แจ้งซ่อม 5 รายการ, ของหาย-ของพบ 3 รายการ, ประกาศ 2 รายการปักหมุด

**สำรอง:** อัดคลิปหน้าจอ 5 นาทีนี้ไว้ล่วงหน้า (ตามแผน 08-07 ใน PRESENTATION-PREP-PLAN.md) เผื่อเน็ต/LINE ล่มวันจริง

---

## ลำดับการสาธิต

### 0:00–0:30 — เปิดแอปจาก LINE → หน้าหลัก
- เปิดจาก Rich Menu หรือลิงก์ LINE OA (หรือพิมพ์ URL ตรงถ้า Rich Menu ยังไม่พร้อม — บอกกรรมการว่า "โครงสร้าง Rich Menu พร้อมแล้ว รอเลข LINE OA")
- ชี้ให้ดู: โปรไฟล์ + สังกัดกองพัน/กองร้อยจริงบนหัวข้อ, กระดิ่งแจ้งเตือนมีตัวเลขจริง (ไม่ใช่ hardcode), ประกาศสำคัญปักหมุดบนสุด
- พูด: "ข้อมูลทั้งหมดต่อฐานข้อมูลจริง ไม่มีข้อมูลปลอมเหลืออยู่แล้ว"

### 0:30–1:15 — ตารางเรียนวันนี้ (`/class`)
- แตะ "ตารางเรียน" จาก Quick services หรือ bottom nav
- เลื่อนดูตารางรายวัน ชี้ badge "วันนี้" ตรงวันจริง, สีตามหมวดวิชา (วิชาการ/ทหาร/พละ/สอน/self-study)
- พูด: "ดึงข้อมูลจริงจากฐานข้อมูล ไม่ใช่ mock 9 สังกัดแบบเดิมแล้ว"

### 1:15–1:45 — เมนูอาหาร (`/meals`)
- แตะเมนูอาหารจาก Quick services
- เลื่อนดูสัปดาห์ปัจจุบัน เช้า/กลางวัน/เย็น
- พูด (ถ้ามีสัปดาห์ไหนว่าง): "ช่วงที่ยังไม่มีเมนูจะโชว์ empty state ที่อ่านง่าย แทนที่จะเป็นขีดว่างๆ ทั้งตาราง"

### 1:45–2:30 — แจ้งซ่อมพร้อมรูป (`/report`)
- เข้าหน้า "แจ้งซ่อม" — ตอนนี้เป็น **list-first** เห็นรายการเก่าก่อนเลย ไม่ใช่ฟอร์มเปล่าเหมือนเดิม
- แตะปุ่ม + (FAB มุมล่างขวา, สไตล์เดียวกับกิจกรรม/ของหาย) → กรอกหัวข้อ+รายละเอียด (บังคับกรอกทั้งคู่) → แนบรูป (แสดง client-side compression) → ปักหมุดสถานที่ → ส่ง
- กลับมาหน้า list เห็นรายการใหม่ทันที แตะดู accordion ประวัติสถานะ
- พูด: "ตัด dropdown เลือกทีมออกจาก UI แล้ว ผู้แจ้งไม่ต้องรู้ว่าใครดูแล เจ้าหน้าที่จัดสรรทีหลัง"

### 2:30–3:15 — กิจกรรม (`/activity`)
- เข้าหน้ากิจกรรม เห็น photo card แนวตั้ง
- เปิดกิจกรรมหนึ่งรายการ → แตะ "ลงทะเบียนเข้าร่วม" → เห็นจำนวนที่นั่งอัปเดตทันที
- (ถ้ามีเวลา) แตะ "ยกเลิกลงทะเบียน" โชว์ flow ยกเลิกได้จริง ไม่ใช่ค้างตลอดไปเหมือนก่อน
- พูด: "กิจกรรมที่สร้างใหม่อนุมัติอัตโนมัติ (auto-approve) เพราะยังไม่มีบัญชี moderator จริง — บันทึกเป็นข้อจำกัดที่ตัดสินใจแล้วในเอกสาร"

### 3:15–4:00 — ของหาย-ของพบ (`/lost-found`)
- เข้าหน้า เห็น filter chip ทั้งหมด/ของหาย/ของพบ ใช้ field จริงแล้ว (ไม่ใช่แปะข้อความ)
- เปิดรายการหนึ่ง ชี้ badge ประเภท + ปุ่ม claim ที่ข้อความเปลี่ยนตามประเภท ("ฉันพบของชิ้นนี้" กับ "นี่คือของฉัน")
- พูด: "ปุ่ม + ย้ายเป็น FAB ลอยมุมล่างขวาแบบเดียวกับหน้าอื่น ให้ UI สม่ำเสมอทั้งแอป"

### 4:00–4:35 — การแจ้งเตือน + ตั้งค่า
- แตะกระดิ่ง → เห็นรายการแจ้งเตือนจริง แตะอันหนึ่ง → อ่านแล้วเปลี่ยนสถานะ + พาไปหน้าที่เกี่ยวข้องอัตโนมัติ (deep link)
- เข้า `/settings` → ชี้สถานะเชื่อมต่อ LINE จริง (ไม่ใช่ "เร็วๆ นี้" แล้ว), สลับธีมมืด/สว่างโชว์ dark mode, แตะนโยบาย PDPA เข้าไปดูเนื้อหาจริง
- พูด: "ทุกหน้าในแอปรองรับ 2 ภาษาไทย/อังกฤษ และ dark theme ทดสอบบนอุปกรณ์จริงแล้ว"

### 4:35–5:00 — ปิดท้าย
- กลับหน้าหลัก สรุปสั้นๆ: "ระบบทั้งหมดต่อฐานข้อมูลจริง ผ่านการตรวจ security 5 หัวข้อ (auth ครบทุก endpoint, ปิดช่องโหว่ IDOR, validate input, ปิด data leak) และแก้บั๊กจากการทดสอบอุปกรณ์จริงแล้ว"
- เชื่อมโยงกลับวัตถุประสงค์วิจัย (ใช้ 1 ประโยคจาก `docs/research/RESEARCH-OBJECTIVE-ALIGNMENT-2026-06-29.md`)
