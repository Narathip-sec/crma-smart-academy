**บริบท:** iOS ทดสอบรอบล่าสุดคือ **ก่อน** Playbook 2 ทั้งหมด (ก่อน security fix, ก่อนต่อ DB จริง, ก่อน UX fix ทุกอย่าง, ก่อน Rich Menu/logo/avatar sync ที่เพิ่งทำไป) — เท่ากับยังไม่เคยทดสอบงานทั้งหมดของ session นี้บน iOS เลย ต้องถือว่าเป็นรอบแรกจริงๆ ไม่ใช่แค่ retest

**วิธีทดสอบ:** เปิดผ่าน LINE จริงบน iPhone (ไม่ใช่ Safari เปล่าๆ — LIFF webview บน iOS คือ WKWebView ซึ่งพฤติกรรมต่างจาก Safari ปกติในบางจุด)

## Code review ทำไปแล้ว — จุดเสี่ยงเฉพาะ iOS ที่ต้องดูตาจริง

- [ ] **Safe-area / home indicator** — โค้ดใช้ `env(safe-area-inset-bottom, 0px)` ใน bottom nav กับ sheet แต่ไม่มี `viewport-fit=cover` ใน viewport meta เลย ซึ่งแปลว่า `env()` น่าจะ resolve เป็น 0 เสมอ (ไม่ได้ดึงค่าจริงจาก notch/home indicator) — เช็คว่า bottom nav หรือ FAB ทับ/ชิดเส้น gesture bar ด้านล่างจอ (iPhone ไม่มีปุ่ม Home) ในหน้า LIFF จริงหรือไม่ ถ้าทับ ต้องกลับมาแก้ตรงนี้
- [ ] **Native photo chooser** — ยืนยันแล้วจากรูปที่ส่งมาว่า Photo Library / Take Photo / Choose File ขึ้นครบ (ปุ่มเดียว ไม่มี capture attribute) — เช็คซ้ำอีกทีหลัง Rich Menu/logo เปลี่ยน ว่ายังทำงานเหมือนเดิม
- [ ] **createImageBitmap (image compression)** — รองรับ Safari 15+ ตามสเปก ควรใช้ได้ปกติ แต่เช็คว่าไฟล์รูปจากกล้อง iPhone (HEIC บีบอัดแล้วเป็น JPEG) อัปโหลดสำเร็จจริง

## ฟีเจอร์ใหม่ทั้งหมดของ session นี้ — ยังไม่เคยทดสอบบน iOS เลย

- [ ] Rich Menu ขนาดใหญ่ขึ้น (2500x1686) — เปิดจาก LINE เห็นเมนูเต็มจอ ปุ่มใหญ่ กดง่ายขึ้นจริงไหม
- [ ] Logo ตราครุฑ/ครูตราใน TopBar — ขึ้นถูกต้อง กดแล้วพากลับหน้าแรก
- [ ] TopBar ไม่มี avatar วงกลมซ้ายมือแล้ว (ตัดออกแล้ว) — เช็คว่า layout ไม่เพี้ยน
- [ ] Avatar sync กับรูป LINE จริง — TopBar, การ์ดโปรไฟล์หน้าแรก, หน้า Profile ทั้ง 3 จุด
- [ ] หน้า Profile: ไม่มี "รหัส" แล้ว, "สังกัด" แสดงแบบ "พัน.X ร้อย.Y"
- [ ] `/class` ตารางเรียนต่อ DB จริง ไม่ซ้ำข้ามสัปดาห์
- [ ] ประกาศ/ข่าว ต่อ DB จริงทั้งระบบ
- [ ] ของหาย/พบ: filter ประเภท, ปุ่ม claim, badge ประเภท+สถานะ, กดขอรับของสำเร็จ (ไม่ error)
- [ ] ของหาย/พบ + กิจกรรม detail: เปิดโพสต์ที่ตัวเองสร้าง เห็นรายชื่อผู้ลงทะเบียน/ผู้ขอรับ, เปิดโพสต์คนอื่นสร้าง เห็นแค่จำนวน
- [ ] ฟอร์มแจ้งซ่อม: ไม่มี dropdown ทีม, `/report` list-first + FAB
- [ ] `/lost-found` FAB (เปลี่ยนจากปุ่ม + เดิม)
- [ ] Settings: สถานะเชื่อมต่อ LINE จริง, กด PDPA เข้าได้
- [ ] กิจกรรม: ลงทะเบียน → ยกเลิกลงทะเบียน → ลงใหม่ ครบ loop, จำนวนที่นั่งอัปเดตถูก
- [ ] เมนูอาหาร: สัปดาห์ไม่มีข้อมูล → empty state (ไม่ใช่ตารางขีดว่าง)
- [ ] กระดิ่งแจ้งเตือน: ตัวเลขตรงจริง, กดอ่านแล้วลดตัวเลข, พาไปหน้าที่เกี่ยวข้อง
