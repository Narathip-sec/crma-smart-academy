เปลี่ยนคำแนะนำเป็น Lost & Found / ของหาย-ของพบ

เหตุผลคือระบบขอลาของ รร.จปร. ยังเป็นกระดาษและไม่น่าจะเปลี่ยนเร็ว การเลือก Lost&Found จึงเหมาะกว่า: ทำได้จริง งบต่ำ ใช้กับชีวิตในโรงเรียนประจำ และต่อยอดกับ LINE notification ได้ง่าย

Cadet

Finder

แจ้งพบของ

LINE

แจ้งเตือนสถานะ

ข้อควรระวัง

รูปภาพอาจติดใบหน้า บัตรประจำตัว หรือเอกสารส่วนบุคคล จึงควรมี moderation และซ่อนรายละเอียดบางส่วนของของสำคัญ เช่น บัตรหรือเอกสารราชการ

Development Phases

Phase 0 - Project Setup

Next.js, TypeScript, Tailwind, Prisma, PostgreSQL, LIFF SDK, app shell, bottom nav, design tokens

Phase 1 - MVP UI + Mock Data

Home, Class, Meals, Calendar, Todo, Service, Profile/Grades โดยใช้ mock data และไม่มี AI

Phase 2 - Workflow Features

Report พร้อม Pin on Map, Lost&Found, Activity Meetup-style, RSVP, event creation

Phase 3 - Backend + Import

Prisma schema, import scripts, database queries, admin import preview, audit log

Phase 4 - LINE Integration + Pilot

LIFF login, LINE user binding, Rich Menu, push notification และ pilot กับกลุ่มเล็ก

Claude Code Prompts

Prompt 1

สร้างโปรเจ็กต์ใหม่

Create a new Next.js App Router project for a LINE LIFF app named CRMA Smart Academy.
Use TypeScript, Tailwind CSS, Prisma, PostgreSQL, and @line/liff.
Do not add any AI feature.
Build a mobile-first app shell with top bar, bottom navigation, and routes:
Home, Class, Todo, Activity, Service, Profile, Meals, Calendar, Report, LostFound.
Prompt 2

หน้า Home

Build the Home page using mock data.
Style: teal brand, light blue background, white rounded cards, avatar header,
featured carousel, quick services, My Day summary, and News list.
Quick services: Grades, Meals, Lost & Found, Reports.
No Health AI and no AI features.
Prompt 3

Class รายวัน

Build the Class page as a fixed daily schedule.
Use Monday-Friday tabs, not a weekly calendar grid.
Data shape: cohort, day, period, start, end, subject code,
subject Thai/English, room, instructor, type.
Prompt 4

Report Map

Build the Report page with category, urgency, description,
photo upload placeholder, and Pin on Map.
Use a campus map image with clickable pin overlay.
Store pin_x and pin_y as percentages.
Do not use paid map APIs.

Checklist ก่อนเริ่ม Build

  • ยืนยันว่า MVP ไม่มี AI ทุกชนิด
  • เลือก feature แทน Health AI: Lost & Found
  • เตรียมไฟล์ตารางเรียนรายวัน
  • เตรียมไฟล์อาหารรายเดือน
  • เตรียมไฟล์ academic calendar รายปี
  • เตรียมรูป campus map สำหรับ Pin on Map
  • เตรียม LINE Official Account และ LINE Developers channel
  • เตรียม LIFF ID และ HTTPS endpoint
  • เตรียม PostgreSQL database
  • เขียน privacy notice และแนวทาง PDPA
  • กำหนดผู้ดูแล event moderation
  • กำหนดผู้ดูแล report ticket และ Lost&Found moderation
