# Android Device Test Checklist (2026-07-21)

**บริบท:** iOS ทดสอบผ่านแล้ว (เจอ+แก้บั๊ก 4 จุด, ดู `PRESENTATION-PREP-PLAN.md`). Android ยังไม่เคยทดสอบเลย และตั้งแต่นั้นมี Playbook 2 เปลี่ยน UI/flow ไปเยอะ (14 task) — checklist นี้ครอบคลุมทั้งของใหม่ทั้งหมด และจุดที่เคยพังบน iOS (เผื่อ Android มี pattern เดียวกัน).

**วิธีทดสอบ:** เปิดผ่าน LINE จริงบน Android (ไม่ใช่ Chrome เปล่าๆ — ต้องผ่าน LIFF webview ถึงจะเจอบั๊กที่เกี่ยวกับ LIFF SDK จริง)

---

## Code review ทำไปแล้ว (ไม่พบบั๊ก แต่ยังต้องดูตาจริงบนอุปกรณ์)

โค้ดที่เสี่ยงต่างกันระหว่าง iOS/Android ตรวจแล้ว ไม่มีจุดที่ branch ผิดแพลตฟอร์มหรือใช้ API เฉพาะ iOS แต่มี 2 พฤติกรรมที่พิสูจน์ด้วยโค้ดอย่างเดียวไม่ได้ ต้องดูจริง:

- [ ] **Viewport height / address bar** — layout ใช้ `min-h-screen` (`100vh`). Android Chrome ที่ address bar ซ่อน/โผล่มักทำให้ความสูงหน้าจอกระโดด — แต่ LIFF webview ปกติไม่มี address bar ให้ซ่อนอยู่แล้ว เช็คว่า bottom nav/FAB ไม่โดนตัดหรือลอยผิดตำแหน่งตอนเลื่อนหน้า
- [ ] **Soft keyboard + fixed FAB** — หน้าที่มี FAB ลอย (`/report`, `/lost-found`, `/activity`) ไม่มีฟอร์มอยู่ในหน้าเดียวกัน (ฟอร์มแยกไปหน้า `/new`) จึงไม่ควรมี FAB ทับ keyboard แต่เช็คหน้าฟอร์มเองว่า input ล่างสุดไม่โดน keyboard บังตอนพิมพ์

## จุดที่เคยพังบน iOS — เช็คซ้ำบน Android

- [ ] ของหาย/ของพบ: รูปขึ้นจริงไหม, badge ประเภท(หาย/พบ)+สถานะแสดงถูก, ปุ่ม claim กดได้ไม่ค้าง
- [ ] กิจกรรม: การ์ดมีรูป, ปุ่ม "ลงทะเบียนเข้าร่วม" กดติด ไม่ค้าง disabled
- [ ] แจ้งซ่อม: list มีรูป/รายละเอียด/สถานที่ครบ, เลขที่ตั๋วขึ้นถูก (ไม่ blank)
- [ ] Settings เข้าถึงง่ายจากหน้าแรก (ไอคอนเห็นชัด)

## ฟีเจอร์ใหม่จาก Playbook 2 — ยังไม่เคยทดสอบบนอุปกรณ์จริงเลย (ทั้ง iOS และ Android)

- [ ] `/class` ตารางเรียนต่อ DB จริง — badge "วันนี้" ตรงวัน, สีตามหมวดวิชาครบ 5 หมวด
- [ ] ประกาศ/ข่าว (home + `/announcements` + detail) ต่อ DB จริง, เวลา "x ชม.ที่แล้ว" อัปเดตถูก
- [ ] ของหาย/ของพบ: filter ทั้งหมด/ของหาย/ของพบ ทำงานถูก, ปุ่ม claim label เปลี่ยนตามประเภท
- [ ] ฟอร์มแจ้งซ่อม: ไม่มี dropdown ทีมแล้ว, กรอกรายละเอียดไม่ครบส่งไม่ได้
- [ ] `/report` list-first: เข้ามาเจอ list ก่อน, FAB มุมล่างขวา → เปิดฟอร์ม → ส่งสำเร็จ → เด้งกลับ list เห็นรายการใหม่
- [ ] `/lost-found` FAB มุมล่างขวา (เปลี่ยนจากปุ่ม + เดิม)
- [ ] Settings: สถานะเชื่อมต่อ LINE ขึ้นจริง (ไม่ใช่ "เร็วๆ นี้"), กด PDPA เข้าไปอ่านได้
- [ ] กิจกรรม: ลงทะเบียนแล้วกด "ยกเลิกลงทะเบียน" ได้จริง, จำนวนที่นั่งอัปเดตถูกทั้งสองทาง
- [ ] เมนูอาหาร: สัปดาห์ที่ไม่มีข้อมูลขึ้น empty state สวยๆ (ไม่ใช่ตารางขีดว่างล้วน)
- [ ] กระดิ่งแจ้งเตือน: ตัวเลขไม่อ่านตรงจำนวนจริง, แตะรายการ → อ่านแล้ว + พาไปหน้าที่เกี่ยวข้อง, ตัวเลขลดลงถูก

## ทั่วไป (มาตรฐานทุกรอบทดสอบอุปกรณ์)

- [ ] Dark theme ทุกหน้า สลับแล้วไม่มีสีตกขอบ/ตัวอ่านไม่ออก
- [ ] อัปโหลดรูปจากกล้อง Android จริง (ไม่ใช่เลือกจากคลัง) — ขนาดไฟล์บีบอัดสำเร็จ ไม่ค้าง
- [ ] Double-submit guard ทุกฟอร์ม (กดส่งรัวๆ ไม่ได้ record ซ้ำ)
- [ ] ปุ่ม back ของ Android (ฮาร์ดแวร์/gesture) ทำงานตามที่คาด ไม่ค้างหรือออกแอปผิดจังหวะ
