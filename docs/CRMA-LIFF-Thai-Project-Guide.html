ภาพรวมโปรเจ็กต์

เป้าหมายคือสร้างเว็บแอปแบบ LIFF ที่เปิดจาก LINE Official Account ได้ทันที สำหรับนักเรียนนายร้อยและผู้เกี่ยวข้อง โดย UI จะเป็น mobile-first เหมือนภาพตัวอย่าง: พื้นหลังสีครีมอุ่น, ตัวอักษรน้ำเงินเข้ม, accent สีแดงเลือดหมู, ขอบบาง, card ไม่โค้งมาก, และ bottom navigation 5 เมนู

คำแนะนำหลัก

เริ่มจาก MVP ที่ใช้จริงได้ก่อน: Home, Calendar, To-do-list, Activity, Service, Profile/Grade, Class Schedule และ LINE login จากนั้นค่อยเพิ่ม Meal AI, Health AI, Report/Fix และ dashboard ภายหลัง

สิ่งที่ต้องเตรียมก่อนเริ่ม

บัญชีและระบบ

LINE / Hosting / Database

  • LINE Official Account ของโครงการหรือหน่วยงาน
  • LINE Developers Provider และ channel สำหรับ LIFF
  • LIFF ID, endpoint URL, channel secret, channel access token
  • Vercel project สำหรับ deploy เว็บ Next.js
  • PostgreSQL เช่น Vercel Postgres, Supabase, Neon หรือฐานข้อมูลของหน่วยงาน
ข้อมูลจริง

ข้อมูลที่ต้องจัดเป็นไฟล์นำเข้า

  • รายชื่อนักเรียนนายร้อย: รหัส, ชั้นปี, กองพัน, กองร้อย, อีเมล
  • รายวิชา, อาจารย์, ห้องเรียน, ตารางเรียน, ภาคการศึกษา
  • ข่าวประกาศ, ปฏิทินกิจกรรม, งานที่มอบหมาย
  • เมนูอาหาร, โภชนาการ, ข้อมูลสุขภาพที่อนุญาตให้ใช้
  • หมวดหมู่แจ้งซ่อม, พื้นที่อาคาร, ผู้รับผิดชอบ, SLA
นโยบาย

ความปลอดภัยและ PDPA

  • Privacy notice ภาษาไทยและอังกฤษ
  • ข้อความขอความยินยอมสำหรับข้อมูลสุขภาพ/ตำแหน่ง/รูปภาพ
  • กำหนดบทบาท: cadet, instructor, command, admin
  • กำหนดข้อมูลที่ห้ามเก็บใน POC เช่นข้อมูลลับหรือข้อมูลอ่อนไหวเกินจำเป็น
  • กำหนด audit log สำหรับการดูเกรด, profile, report และ admin action
งานออกแบบ

Asset และ UI System

  • โลโก้ CRMA และไอคอนที่ได้รับอนุญาต
  • ภาพประกาศ/กิจกรรมที่เหมาะกับหน้าจอมือถือ
  • ชุดสีหลัก: navy, crimson, gold, paper, bone
  • ข้อความทุกหน้าทั้งภาษาไทยและอังกฤษ
  • สถานะว่าง, โหลดข้อมูล, error, ไม่มีสิทธิ์เข้าถึง

ควรเริ่มจากตรงไหน

กำหนด MVP ให้เล็กและชัด

เวอร์ชันแรกไม่ควรพยายามทำทุกอย่างจากภาพพร้อมกัน ให้เริ่มจากหน้าที่นักเรียนนายร้อยเปิดใช้ทุกวัน: Home, Calendar, To-do-list, Class, Profile/Grade และ Service

ทำ UI ให้ตรง mood ของภาพก่อน

สร้าง design tokens, shell, bottom nav, app bar, card, chip, placeholder image และ bilingual label ให้แน่นก่อน แล้วค่อยทำแต่ละหน้า จะทำให้โปรเจ็กต์ดูเป็นระบบตั้งแต่แรก

เชื่อม LIFF แบบปลอดภัย

ใช้ LIFF SDK เพื่อเริ่ม session และอ่าน profile เท่าที่จำเป็น จากนั้น bind LINE user ID กับบัญชีโรงเรียนหรือรหัสนักเรียนนายร้อย อย่านำ channel secret ไปใส่ฝั่ง browser

เปลี่ยน mock data เป็นฐานข้อมูลทีละส่วน

เริ่มจากตาราง cadet, course, enrollment, schedule, grade, announcement และ calendar event ให้ UI ใช้ mock data ได้ก่อน แล้วค่อยเปลี่ยนเป็น Prisma query

ข้อควรระวังของ LIFF

Endpoint URL ต้องเป็น HTTPS เมื่อใช้งานจริง, ต้องเรียก liff.init() ก่อนใช้ LIFF API, และหากทำเป็น SPA ให้ใช้ History API แทนการพึ่ง URL fragment มากเกินไป

  • Design tokens ใน CSS variables
  • Component: AppBar, Card, Chip, Metric, ImagePlaceholder, EmptyState
  • Calendar page และ To-do-list page แบบใช้ mock data
  • Service search และ Activity cards
Backend

สิ่งที่ต้องตามมา

  • Role-based access control
  • Prisma schema สำหรับ schedule, grade, calendar, task, announcement
  • Admin import CSV และ content publish workflow
  • LINE Messaging API สำหรับ push notification

Prompt สำหรับใช้กับ Claude Code

ใช้ prompt เหล่านี้ทีละข้อ จะควบคุม scope ได้ง่ายกว่าให้ Claude Code ทำทุกอย่างพร้อมกัน

Prompt 1: ตรวจโปรเจ็กต์และวางแผน

Read CLAUDE.md, web/README.md, and the existing Next.js app.
Summarize current implementation status, identify placeholder screens,
then propose a Phase 1 plan for Calendar, To-do-list, Service, Activity,
and LIFF login. Do not edit files yet.

Prompt 2: ทำ Calendar page

Implement the Calendar page to match the supplied mobile screenshots:
cream paper background, navy text, crimson active date, gold/navy event dots,
selected-day agenda, and fixed bottom nav. Use existing project components
and mock data. Run lint/build after editing.

Prompt 3: ทำ To-do-list page

Replace the To-do-list placeholder with a production-quality mobile UI:
summary metrics, category chips, checkbox rows, overdue/urgent badge,
completed state, and Thai/English copy using the existing i18n pattern.
Use mock data first, no backend yet.

Prompt 4: เชื่อม LIFF แบบปลอดภัย

Review src/lib/liff.ts and implement a safe LIFF initialization flow.
Use NEXT_PUBLIC_LIFF_ID only for the public LIFF ID.
Do not expose channel secret or access token to the browser.
Add fallback behavior for local development and explain required env vars.

Prompt 5: ขยาย Prisma schema

Extend prisma/schema.prisma for Phase 1:
User, LineAccount, RoleAssignment, Announcement, CalendarEvent, Task,
TaskAssignment, Notification, NotificationDelivery, AuditLog.
Keep existing Cadet/Course/Grade/Schedule models compatible.
Generate migration notes and seed-data plan.

คำแนะนำให้โปรเจ็กต์ออกมาดี

UX

ทำให้ใช้ทุกวันได้จริง

  • หน้าแรกต้องตอบคำถาม “วันนี้ต้องทำอะไร” ภายใน 5 วินาที
  • ให้ notification เป็นแบบ personal และมีประโยชน์ ไม่ใช่ broadcast ทุกเรื่อง
  • ทุกหน้าต้องมี empty state และ error state ที่สุภาพ
  • Thai text ต้องไม่ล้น card หรือปุ่มบนจอมือถือแคบ
Product

อย่าข้ามระบบหลังบ้าน

  • ถ้าไม่มี admin workflow ข้อมูลจะเก่าเร็ว และผู้ใช้จะเลิกเชื่อแอป
  • กำหนดเจ้าของข้อมูลแต่ละส่วน เช่น ข่าว ตารางเรียน เกรด กิจกรรม
  • มี audit log ตั้งแต่แรก โดยเฉพาะข้อมูลเกรดและข้อมูลส่วนบุคคล
  • วัดผล pilot ด้วย DAU, open rate, task completion, report SLA และ feedback
Security

เก็บความลับให้ถูกที่

  • ค่าที่ขึ้นต้น NEXT_PUBLIC_ ถือว่าเห็นได้จาก browser
  • channel secret, access token, database URL ต้องอยู่ฝั่ง server เท่านั้น
  • จำกัดสิทธิ์ test phase เฉพาะผู้ใช้ pilot
  • แยกข้อมูลสุขภาพและตำแหน่งออกจากข้อมูลทั่วไป พร้อม consent ชัดเจน
Delivery

ทำทีละรอบเล็ก ๆ

  • ทำหนึ่งหน้าให้จบทั้ง responsive, i18n, empty/error state ก่อนข้ามไปหน้าใหม่
  • หลังแก้ทุกครั้งให้ run lint/build และเปิดดูบนจอมือถือ
  • เก็บ screenshot ก่อน/หลัง เพื่อเทียบกับภาพต้นแบบ
  • อย่าเริ่ม AI health/report automation ก่อน privacy และ data owner ชัด

Checklist ก่อนเปิด pilot

หมวด ต้องผ่านก่อน pilot หมายเหตุ
LIFF เปิดจาก https://liff.line.me/{liffId} ได้, endpoint เป็น HTTPS, login สำเร็จ ทดสอบทั้งใน LINE app และ external browser
ข้อมูล มี seed/import สำหรับ cadet, schedule, grade, calendar, task ใช้ข้อมูลจำลองก่อน หากยังไม่ผ่านอนุมัติข้อมูลจริง
สิทธิ์ ผู้ใช้เห็นเฉพาะข้อมูลของตัวเองหรือกลุ่มที่มีสิทธิ์ ตรวจทั้ง cadet, instructor, admin
PDPA มี privacy notice, consent, audit log, data retention policy ให้ฝ่ายกฎหมายหรือ DPO ตรวจ
UX ข้อความไทยไม่ล้น, ปุ่มกดง่าย, loading/error ชัด, bottom nav ไม่บังเนื้อหา ทดสอบบน iPhone/Android จริง

แหล่งอ้างอิงสำคัญ

  • LINE Developers: Create a channel อธิบายว่าต้องสร้าง provider/channel และ LIFF เพิ่มได้กับ LINE Login หรือ LINE MINI App
  • LINE Developers: Adding a LIFF app to your channel สำหรับ LIFF ID, scope, endpoint URL และการ redirect
  • LINE Developers: Developing a LIFF app สำหรับ LIFF SDK, liff.init(), profile และ LIFF API
  • LINE Developers: LIFF app development guidelines สำหรับ security, HTTPS, routing และข้อควรระวังเรื่อง user data
